📧 4 — Email setup Not started ▾
Enable Cloudflare Email Routing
Cloudflare → domain → Email → Email Routing → Enable. Cloudflare will add the required MX records automatically.
Cloudflare ↗
Create email forwarding rules
Cloudflare → Email Routing → Routing rules → Create address. Set up info@, hello@, or whatever aliases the client needs. Forward each to their personal inbox.
Add SPF record
Cloudflare Email Routing adds an SPF record automatically when enabled. Verify it exists in DNS → TXT records. Should contain "include:_spf.mx.cloudflare.net".
Add DMARC record
In Cloudflare DNS add a TXT record: Name: _dmarc, Value: "v=DMARC1; p=none; rua=mailto:info@theirdomain.com"
Set up Gmail Send As (for client)
Help the client add their custom address as a Gmail Send As alias so they can reply from info@theirdomain.com. Gmail → Settings → Accounts → Add another email address → use smtp.gmail.com with their Gmail credentials.
